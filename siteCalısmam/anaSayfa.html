Most Agile ideas originated in the 1970s. Studies and examinations have been made about the Agile Method, which emerged as a reaction against traditional project approaches. In 1970, Dr. William Royce has outlined his specific ideas about sequential development by publishing an article discussing the management and development of large software systems. Royce emphasized that a project can be developed like a product on an assembly line. He drew attention to the idea that each stage of development must be completed before the next stage begins. He emphasized that all developers should bring together all the requirements of a project. He stated that the next step is to complete the entire architecture and designs of the project. He emphasized that when these steps were completed, there was little or no contact between the expert groups that completed each phase of the project. Therefore, the pioneers of the agile method believed that if project developers studied the process, they would find it the most logical and useful solution to software development........

Structured Query Language, abbreviated as SQL is a domain-specific language used in programming and designed for managing data held in a relational database management system (RDBMS), or for stream processing in a relational data stream management system (RDSMS). It is particularly useful in handling structured data, i.e. data incorporating relations among entities and variables. SQL offers two main advantages over older read–write APIs such as ISAM or VSAM. Firstly, it introduced the concept of accessing many records with one single command. Secondly, it eliminates the need to specify how to reach a record, e.g. with or without an index. Originally based upon relational algebra and tuple relational calculus, SQL c onsists of many types of statements, which may be informally classed as sublanguages, commonly: a data query language (DQL), a data definition language (DDL), a data control language (DCL), and a data manipulation language (DML). The scope of SQL includes data query, data manipulation (insert, update, and delete), data definition (schema creation and modification), and data access control.
